\chapter*{Хураангуй}
\addcontentsline{toc}{chapter}{Хураангуй}

Энэхүү дипломын ажил нь нийгмийн сүлжээний нээлттэй олон нийтийн өгөгдлийг автоматаар цуглуулж, тэдгээрт Sentiment Analysis хийж, хэрэглэгчид хэрхэн оролцоо үзүүлэх болон ямар төвөгтэй хариу үйлдэл авах магадлалыг таамаглах системийг боловсруулахад чиглэсэн болно. Судалгааны практик хэрэгжилт нь нийгмийн сүлжээний өгөгдөлд төвлөрсөн бөгөөд модульчлагдсан архитектур нь ирээдүйд олон төрлийн эх сурвалж нэмэх боломжтой байхаар зохион байгуулагдсан.

\textbf{Судалгааны үндсэн зорилгууд:}
\begin{itemize}
    \item Нийгмийн сүлжээний хуудаснаас нийтлэл, сэтгэгдлийг автоматаар цуглуулах систем бүтээх
    \item Цуглуулсан мэдээлэлд байгалийн хэлийн боловсруулалт ашиглан Sentiment Analysis хийх
    \item Мэдрэмж, оролцооны метрик, цаг хугацааны өгөгдөлд тулгуурлан хүмүүсийн оролцоотой байх магадлал болон оролцооны хэмжээг таамаглах
    \item Таамагласан үр дүнд үндэслэн автомат зөвлөмж үүсгэх алгоритм боловсруулах
    \item Системийг хэрэглэхэд хялбар болгох интерфейс.nашиглах
    \item Цуглуулсан өгөгдлийг безопаснаар хадгалах ба удирдах
\end{itemize}

Систем нь өгөгдлийн цуглуулга, боловсруулалт, Sentiment Analysis, таамаглалт болон зөвлөмжийн модультай байна. Мэдрэмжийн шинжилгээнд бүтээлэг байгалийн хэлийн боловсруулалтын аргуудыг ашигласан. Туршилтын үр дүнд систем нь нийгмийн сүлжээний хуудаснаас тогтвортой өгөгдөл цуглуулж, Sentiment Analysis-ын нарийвчлал 91\%-д хүрсэн бөгөөд оролцоотой байхуу болон оролцооны хэмжээний таамаглалд суурилсан зөвлөмжийн практик хамаарал 87\% гарсан.

\textbf{Түлхүүр үгс:} Нийгмийн сүлжээ, вэб цуглуулга, sentiment analysis, оролцооны таамаглал, байгалийн хэлийн боловсруулалт, сурсан машин, таамаглалтын загвар

\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

This thesis presents an automated system for collecting social media public data, performing sentiment analysis, and estimating how users are likely to engage and how much engagement a post may receive. The practical implementation focuses on social media platforms, while the modular architecture keeps the system extensible for future data sources. The system uses automated data collection techniques to gather posts and comments, applies natural language processing (NLP) for sentiment analysis, and combines sentiment, engagement, and temporal signals to estimate engagement likelihood and engagement volume, then generate recommendations based on analysis.

\textbf{The system's main objectives are:}
\begin{itemize}
    \item Develop an automated data collection system for public social media content
    \item Implement sentiment analysis using natural language processing techniques
    \item Predict likely user engagement and engagement volume using sentiment, engagement metrics, and temporal patterns
    \item Develop automated recommendation generation algorithms based on predicted outcomes
    \item Create a user-friendly interface
    \item Store and manage collected data securely
\end{itemize}

The system comprises four major modules: data collection, data processing, sentiment analysis, and engagement prediction with recommendation generation. The sentiment analysis component employs advanced natural language processing methods. The prediction component combines machine learning techniques with sentiment signals, engagement metrics, temporal patterns, and topic clustering to estimate engagement likelihood (active/inactive/neutral) and expected engagement volume, then generate actionable insights.

Experimental results show that the system can reliably collect public social media content with sentiment analysis accuracy of 91\% and engagement prediction/recommendation relevance of 87\%.

\textbf{Keywords:} Social Media, Data Collection, Sentiment Analysis, Engagement Prediction, Natural Language Processing, Machine Learning, Predictive Modeling

Систем нь нийгмийн сүлжээний өгөгдөл дээр баталгаажсан бөгөөд архитектур нь дараагийн шатанд бусад төрлийн эх сурвалж нэмэхээр өргөтгөх боломжтой.

\textbf{Систем юу хийх ёстой вэ?}
\begin{itemize}
    \item Нийгмийн сүлжээний хуудаснаас нийтлэл, сэтгэгдлийг автоматаар цуглуулах
    \item Цуглуулсан текст өгөгдөлд Sentiment Analysis хийж хандлагыг тодорхойлох
    \item Мэдрэмж, оролцоо, хугацааны хэв маягт тулгуурлан контентийн оролцооны магадлал болон оролцааны хэмжээг таамаглах
    \item Таамаглалд суурилсан практик зөвлөмжийг хэрэглэгчид ашиглахад хялбар байдлаар гаргах
\end{itemize}

\textbf{Дэд зорилтууд:}
\begin{enumerate}
    \item Нийгмийн сүлжээний нийтлэл, сэтгэгдлийг цуглуулах найдвартай систем бүтээх
    \item Цуглуулсан өгөгдлийг үр дүнтэйгээр боловсруулах
    \item Sentiment Analysis-ын нарийвчлалыг сайжруулах зөндөө байгалийн хэлийн боловсруулалтын арга ашиглах
    \item Мэдрэмж, оролцоо, хугацааны өгөгдөлд суурилан оролцооны магадлал ба оролцааны хэмжээний таамаглал гаргах
    \item Сурсан машины аргууд ашиглан таамаглалтын нарийвчлалыг сайжруулах
    \item Хэрэглэгчид ашиглахад хялбар системийн интерфейс бүтээх
    \item Цуглуулсан өгөгдлийг безопаснаар хадгалах ба удирдах системийг бүтээх

    \item Динамик контенттой ажиллах
\end{itemize}


\textbf{Яагаад ашигласан бэ?} Selenium WebDriver нь JavaScript-аар ачаалагддаг динамик хуудсуудтай ажиллах, нэвтрэх үйлдэл гүйцэтгэх, хүний үйлдлийг дуурайлган тогтвортой скрапинг хийх боломж олгодог тул сонгосон.

\textbf{Хаана ашигласан бэ?} Энэ технологийг системийн BrowserManager болон PostScraper бүрэлдэхүүнүүдэд ашиглаж, Facebook хуудас руу нэвтрэх, navigate хийх, постын элементүүдийг олох, сэтгэгдлийн modal нээхэд хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Нийгмийн сүлжээний веб интерфэйсийг автоматжуулж, контент ачаалах, interaction хийх, скрапингийн урсгалыг ажиллуулахад ашиглана.

\subsection{BeautifulSoup}

BeautifulSoup нь HTML/XML баримт бичгийг задлан шинжлэх Python номын сан юм.


\textbf{Яагаад ашигласан бэ?} BeautifulSoup нь HTML бүтцийг энгийн, ойлгомжтой аргаар задлан шинжлэх боломжтой бөгөөд Selenium-оор авсан page source-оос цэвэр текстэн өгөгдөл гаргаж авахад тохиромжтой.

\textbf{Хаана ашигласан бэ?} Энэ технологийг PostScraper болон comment extraction логикт ашиглаж, modal болон хуудасны HTML-ээс нийтлэл, сэтгэгдэл, зохиогчийн мэдээллийг ялган авахад хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} HTML/XML баримт бичгээс шаардлагатай өгөгдлийг selector болон tag бүтэц ашиглан задлан авахад ашиглана.

\section{Natural Language Processing (NLP)}

\subsection{NLP-ийн тухай ойлголт}

Natural Language Processing (NLP) нь компьютерт хүний хэлийг ойлгуулах, боловсруулах, үүсгэх чадварыг олгодог хиймэл оюун ухааны салбар юм.

NLP-ийн үндсэн даалгаврууд:
\begin{itemize}
    \item Текст ангилалт (Text Classification)
    \item Sentiment Analysis
    \item Нэрт объект таних (Named Entity Recognition)
    \item Машин орчуулга (Machine Translation)
    \item Асуултанд хариулах (Question Answering)
\end{itemize}

\textbf{Яагаад ашигласан бэ?} Нийгмийн сүлжээний өгөгдөл нь голчлон чөлөөт хэлбэрийн текст байдаг тул түүнийг автоматаар ойлгох, ангилах, утга ба хандлагыг нь тодорхойлохын тулд NLP шаардлагатай.

\textbf{Хаана ашигласан бэ?} NLP-ийг DataAnalyzer, AdvancedSentimentAnalyzer, AIAnalyzer модулиудад ашиглаж, пост болон сэтгэгдлийн агуулгыг боловсруулан мэдрэмж, сэдэв, хэв маягийг тодорхойлоход хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Текст өгөгдлийг цэвэрлэх, ангилах, шинжлэх, түүнээс шийдвэр гаргалтад хэрэгтэй мэдээлэл гаргаж авахад ашиглана.

\subsection{Sentiment Analysis}

Sentiment Analysis нь текстээс хүний мэдрэмж, санал бодлыг илрүүлэх процесс юм. Гурван үндсэн ангилал:

\begin{itemize}
    \item \textbf{Эерэг (Positive):} Сэтгэл ханамжтай, баярлаж буй
    \item \textbf{Сөрөг (Negative):} Сэтгэл дундуур, уурлаж буй
    \item \textbf{Төвийг сахисан (Neutral):} Тодорхой мэдрэмж илрэхгүй
\end{itemize}

\textbf{Яагаад ашигласан бэ?} Хэрэглэгчдийн сэтгэгдэл, санал бодол, брэндийн талаарх хандлагыг тоон хэлбэрт оруулж дүгнэхийн тулд Sentiment Analysis хэрэгтэй.

\textbf{Хаана ашигласан бэ?} Энэ аргыг цуглуулсан нийтлэл, сэтгэгдлийн текст дээр ажиллуулж, DataAnalyzer болон AIAnalyzer доторх Sentiment Analysis pipeline-д ашигласан.

\textbf{Үндсэн хэрэглээ} Текстийг эерэг, сөрөг, төвийг сахисан ангилалд хувааж, нийт хандлагын дүр зургийг гаргахад ашиглана.

\textbf{VADER} (Valence Aware Dictionary and sEntiment Reasoner) нь нийгмийн сүлжээний текстэд тохирсон Sentiment Analysis-ын хэрэгсэл юм:

\subsection{BERT Transformer Model}

BERT (Bidirectional Encoder Representations from Transformers) нь Google-ийн боловсруулсан урьдчилсан сургалттай хэлний загвар юм. BERT нь контекстийг хоёр чиглэлд уншиж, илүү нарийвчлалтай NLP даалгавруудыг гүйцэтгэдэг.


\textbf{Яагаад ашигласан бэ?} BERT нь контекстийг хоёр чиглэлээс ойлгож чаддаг тул уламжлалт rule-based болон lexicon-based аргуудаас илүү нарийвчлалтай Sentiment Analysis хийх боломж олгодог.

\textbf{Хаана ашигласан бэ?} Энэ загварыг AdvancedSentimentAnalyzer болон AIAnalyzer модульд ашиглаж, өндөр чанарын sentiment classification болон confidence score тооцоход хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Илүү нарийн контекст ойлголт шаардсан пост, сэтгэгдлийн өндөр нарийвчлалтай мэдрэмжийн ангилал хийхэд ашиглана.

\section{Өгөгдлийн сан ба API}

\subsection{PostgreSQL}

PostgreSQL нь нээлттэй эхийн, өндөр хүчин чадалтай харилцааны өгөгдлийн сангийн систем юм. Онцлог шинж чанарууд:

\begin{itemize}
    \item ACID (Atomicity, Consistency, Isolation, Durability) дэмждэг
    \item JSON/JSONB өгөгдлийн төрөл
    \item Full-text search
    \item Extensibility
\end{itemize}

\textbf{Яагаад ашигласан бэ?} PostgreSQL нь найдвартай transactional ажиллагаа, бүтэцтэй өгөгдөл хадгалах чадвар, өргөтгөх боломж сайтай тул үндсэн өгөгдлийн сангаар сонгосон.

\textbf{Хаана ашигласан бэ?} Системийн DatabaseManager болон ORM model-ууд дээр ашиглаж, нийтлэл, сэтгэгдэл, шинжилгээний үр дүн, metadata-г хадгалах үндсэн persistent storage болгон хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Production орчинд бүтэцтэй өгөгдлийг хадгалах, query хийх, тайлан болон анализын өгөгдлийг тогтвортой удирдахад ашиглана.

\subsection{Firebase Realtime Database}

Firebase нь Google-ийн cloud-based backend-as-a-service платформ юм. Ашиглах давуу талууд:

\begin{itemize}
    \item Бодит цаг хугацааны синхрончлол
    \item Offline дэмжлэг
    \item Scalability
    \item Authentication
\end{itemize}

\textbf{Яагаад ашигласан бэ?} Firebase нь cloud орчинд хурдан синхрончлол хийх, remote backup хадгалах, lightweight realtime integration хэрэгжүүлэхэд тохиромжтой.

\textbf{Хаана ашигласан бэ?} Энэ технологийг FirebaseDB болон cloud sync логикт ашиглаж, локал өгөгдлийн сантай хамт өгөгдлийг нөөцлөх, синхрончлох зорилгоор хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Realtime sync, cloud backup, distributed access шаардсан үед secondary storage болон integration layer хэлбэрээр ашиглана.

\subsection{REST API ба FastAPI}

REST (Representational State Transfer) нь вэб сервисийн архитектурын загвар юм. FastAPI нь Python-ийн орчин үеийн, хурдан вэб фреймворк юм.


\textbf{Яагаад ашигласан бэ?} FastAPI нь хурдан гүйцэтгэлтэй, async дэмжлэгтэй, type hint дээр суурилсан validation болон автомат API documentation өгдөг тул системийн сервис давхаргад тохиромжтой.

\textbf{Хаана ашигласан бэ?} API server хэсэгт ашиглаж, скрапинг эхлүүлэх, Sentiment Analysis хийх, хуудсуудын жагсаалт авах, экспортын хүсэлт боловсруулах endpoint-уудыг хэрэгжүүлэхэд хэрэглэсэн.

\textbf{Үндсэн хэрэглээ} Системийн боломжуудыг REST endpoint хэлбэрээр гаднын клиент, frontend, automation script-д нээж өгөхөд ашиглана.

\section{Өгөгдлийн загвар}

\subsection{Өгөгдлийн бүтэц}

Системийн өгөгдлийн бүтэц нь дараах энтитүүдээс бүрдэнэ:

\begin{itemize}

    \item \textbf{PostImage:} id (PK), post\_id (FK), image\_url, local\_path, downloaded\_at
    \item \textbf{PostComment:} id (PK), post\_id (FK), comment\_id, author\_name, author\_url, content, timestamp, likes, reply\_to\_id (FK), scraped\_at
\end{itemize}

Харьцаа: FacebookPost $\rightarrow$ PostImage (1:N), FacebookPost $\rightarrow$ PostComment (1:N).

\subsection{Хүснэгтийн бүтэц}

\textbf{Өгөгдлийн загварын хэрэгжилтийн зарчим:} `FacebookPost` хүснэгт нь үндсэн нэгж бөгөөд постын текст, оролцооны метрик (likes, shares, comment\_count), хугацааны тэмдэглэгээ зэрэг үндсэн үзүүлэлтүүдийг хадгална. `PostComment` хүснэгт нь post\_id-гаар холбогдож сэтгэгдлийн агуулга болон reply\_to\_id бүтэцтэйгээр хэлхээт харилцааг хадгалдаг. `PostImage` хүснэгт нь посттой холбоотой медиа URL болон локал хадгалалтын мэдээллийг бүртгэнэ. Энэ бүтэц нь API-гаар өгөгдөл буцаах болон AI шинжилгээ хийх үед нэг ижил дата суурь ашиглах боломжийг бүрдүүлсэн.

\subsection{Sentiment Analysis алгоритм}

Sentiment Analysis-ын урсгалыг дараах дарааллаар хэрэгжүүлсэн:
\begin{enumerate}
    \item Текст цэвэрлэгээ хийх (илүүдэл тэмдэгт, URL, whitespace normalizing)
    \item Суурь анализ (TextBlob/VADER)-аар polarity болон sentiment label тооцоолох
    \item Шаардлагатай нөхцөлд BERT суваг ажиллуулж өндөр нарийвчлалтай ангилал гаргах
    \item Пост ба сэтгэгдлийн түвшний оноог нэгтгэн ерөнхий хандлага тооцох
    \item Гарсан үзүүлэлтүүдийг AIAnalyzer-д дамжуулж хариу үйлдлийн чиглэл, реакцын хэмжээний үнэлгээнд ашиглах
\end{enumerate}

\subsection{Predictive Engagement Modeling алгоритм}

Оролцооны урьдчилсан таамаглалыг дараах урсгалаар боловсруулсан:
\begin{enumerate}
    \item Пост бүрээс feature үүсгэх (content length, word count, hashtag/mention/url count, punctuation, media flag, цаг хугацааны feature, sentiment score)
    \item Хариу хувьсагч болгон engagement score = likes + 2 $\times$ comments + 3 $\times$ shares томьёогоор тооцоолох
    \item RandomForestRegressor ашиглан реакцын хэмжээг regression байдлаар сургах
    \item 75-р хувийн перцентиль босгоор high-engagement ангилал үүсгэж RandomForestClassifier-аар магадлал тооцоолох
    \item R\textsuperscript{2}, MAE, Accuracy, F1 үзүүлэлтээр чанар үнэлж feature importance гаргах
\end{enumerate}

\subsection{Emotion ба Interaction Network анализын алгоритм}

Сэтгэгдлийн зан төлөв болон харилцааны бүтцийг дараах байдлаар үнэлсэн:
\begin{enumerate}
    \item Пост, сэтгэгдлийн текстийг emotion lexicon-оор joy/anger/sadness/fear/surprise/neutral ангилалд оноох
    \item Comment-ийн reply\_to\_id холбоосоор interaction graph үүсгэх
    \item Node, edge, density, degree centrality тооцоолж топ influencer-уудыг тодорхойлох
    \item Community bucket болон dominant emotion-ийг AI зөвлөмж үүсгэх шатанд нэгтгэн ашиглах
\end{enumerate}

\chapter{Хэрэгжилт}

Энэ бүлэгт системийн хэрэгжилтийг судалгааны зорилготой уялдуулан тайлбарлав. Бүрэн эх код нь локал репозиторид хадгалагдсан бөгөөд энэхүү баримт бичигт программын синтакс бус, хэрэгжүүлэлтийн шийдэл, модуль хоорондын уялдаа, AI шинжилгээний урсгалд төвлөрсөн тайлбар өгсөн.

\section{Төслийн бүтэц}

Төслийн хэрэгжилт нь `core`, `ml`, `db`, `api`, `integrations`, `utils` гэсэн үндсэн сангуудтай модульчлагдсан бүтэцтэй. `run.py` нь нэг цэгийн оролтын үүрэг гүйцэтгэж скрапинг, API сервер, AI шинжилгээ зэрэг горимуудыг удирдана. Энэ зохион байгуулалт нь судалгааны туршилтыг дахин гүйцэтгэх, бүрэлдэхүүн хэсгүүдийг тусад нь шалгах боломжийг нэмэгдүүлсэн.


\section{Өгөгдөл Цуглуулах Модуль - Мультиплатформ Дэмжлэг}

Нийгмийн сүлжээнээс өгөгдөл цуглуулах ажлыг илүү мэдэлэл сайтар, уян хатан болгохын тулд мультиплатформ дэмжлэгтэй үйлчилгээ-чиглэл модуль (Data Collection Module) боловсруулсан. Энэхүү модуль нь Facebook, Twitter (X), Instagram гэсэн үндсэн платформуудаас жигд интерфейсээр өгөгдөл авах боломжийг олгодог.

\subsection{Архитектур}

Өгөгдөл цуглуулах модульн архитектур дараах үндсэн бүрэлдэхүүнүүдээс бүрдсэн:

\begin{enumerate}
    \item \textbf{AbstractPlatformCollector:} Платформ-ангит цуглуулагчийн үндсэн интерфейс
    \item \textbf{FacebookCollector:} Facebook платформ-ангит хэрэгжилт (Selenium + PostScraper ашиглалт)
    \item \textbf{TwitterCollector:} Twitter платформ-ангит хэрэгжилт (Twitter API v2 ашиглалт)
    \item \textbf{InstagramCollector:} Instagram платформ-ангит хэрэгжилт (Instagram API ашиглалт)
    \item \textbf{PlatformFactory:} Платформ-ангит цуглуулагчийг үүсгэх угсралт
    \item \textbf{DataCollectionManager:} Олон платформ дээрх цуглуулалтыг удирдах өндөр түвшний менежер
\end{enumerate}

Энэхүү архитектур нь Strategy ба Factory design pattern-үүд ашиглан платформ хоорондын ялгаа нуусан нөхцөлд жигд API үзүүлэхэд үндэс болсон.

\subsection{Цуглуулалтын Параметр}

Өгөгдөл цуглуулалтыг дараах параметрүүдээр тохируулж болно:

\begin{itemize}
    \item \textbf{Платформ:} Facebook, Twitter, Instagram-ийн аль нэгийг сонгох
    \item \textbf{Түлхүүр үгс (Keywords):} Нэг буюу түүнээс олон түлхүүр үгээр хайлт хийх
    \item \textbf{Хугацааны мужих (Date Range):} start\_date болон end\_date-аар хугацаа нарийвчлалтай тохируулах
    \item \textbf{Нийтлэлийн сан (max\_posts):} Цуглуулах нийтлэлийн дээд хязгаар
    \item \textbf{Сэтгэгдлийн сан (max\_comments\_per\_post):} Нийтлэл бүрийн сэтгэгдлийн дээд хязгаар
    \item \textbf{Ашиглалтын тохиромж:} collect\_interactions (реакци, шэйр цуглуулах), collect\_user\_data (хэрэглэгчийн мэдээлэл цуглуулах)
    \item \textbf{Хэл:} Шүүлтүүрлэхийн тулд хэл сонгох
\end{itemize}

Өгөгдөл цуглуулалтаас гарах өгөгдлүүдийг бүтэцлэгдсэн форматаар хадгална:

\subsection{Платформ-ангит Хэрэгжилт}

\subsubsection{Facebook Collector}

Facebook-ын цуглуулалт Selenium браузер менежер болон PostScraper-ийг ашиглане. Хайлт URL ашиглан `https://www.facebook.com/search/posts/?q={keyword}` руу шилжиж нийтлэлүүдийг олж авна. Сэтгэгдлүүдийг өгөгдлийн сангийн загварууд дээрх query-ээр авна.

\subsubsection{Twitter Collector}

Twitter платформ-ангит цуглуулалт Twitter API v2-тай интеграци хийдэг. Хайлтанд `GET /tweets/search/recent` эсвэл `GET /tweets/search/all` endpoint-ыг ашиглан start\_time болон end\_time параметрүүдээр хугацаа нарийвчлалтай шүүлтэрлүүлнэ.

\subsubsection{Instagram Collector}

Instagram платформ-ангит цуглуулалт Instagram API эсвэл instagrapi сангийг ашиглан hashtag хайлт, timeline цуглуулалт хийнэ. Instagram нь қилан-таксины урсалсан хайлтаас ялгаатай төвшөөр утга үг-ын хайлт дэмжихгүй учираас hashtag-аар цуглуулалт хийнэ.

\subsection{Цуглуулалтын Ажиллагаа}

Цуглуулалтын төв урсгал \texttt{DataCollectionManager.collect\_from\_platforms()} арга дараах үйлдлүүдийг дараалан гүйцэтгэнэ:

\begin{enumerate}
    \item Параметр баталгаажуулах (keywords эсвэл хугацаа эсэх, мужихүүдийн буруугүй эсэх)
    \item Платформ бүрийн хувьд PlatformFactory ашиглан цуглуулагчийг үүсгэх
    \item Цуглуулагч тус бүрийн `collect()` арга дуудах
    \item Нийтлэл цуглуулалт: Keywords-ээр хайлт эсвэл хугацаа мужаар timeline цуглуулалт
    \item Хайлтын үр дүнг нэгтгэх ба max\_posts хязгаараар тайрах
    \item Реакци цуглуулалт: collect\_interactions=True үе шатанд коммент, лайк, шэйр гэх мэт реакцуудыг авах
    \item Цуглуулсан өгөгдлийг CollectedPost, CollectedComment объектуудаар нэгтгэх
    \item CollectionResult үр дүн объект буцаах (posts\_collected, comments\_collected, errors гэх мэт)
\end{enumerate}

\subsection{Өгөгдлийн Гаргалт ба Хадгалалт}

Цуглуулсан өгөгдлүүдийг дараах форматаар гаргаж болно:

\begin{itemize}
    \item \textbf{JSON:} \texttt{export\_data()} арга ашиглан \texttt{/posts}, \texttt{/comments}, \texttt{/summary} гэсэн салбаруудтай JSON объект гаргалт
    \item \textbf{Өгөгдлийн сан:} Өгөгдлийг PostgreSQL/SQLite-д хадгалах (existing database layer-ээр хөргөгдөх)
    \item \textbf{Google Sheets:} Integrations/google\_sheets.py ашиглан Google Sheets-д түгээх
\end{itemize}

\subsection{Жишээ Хэрэглээ}

Өгөгдөл цуглуулах модульыг Python кодоор дараах байдлаар ашиглаж болно:

\begin{verbatim}
from core.data_collection_module import (
    DataCollectionManager,
    Platform
)
from datetime import datetime, timedelta

# Менежер үүсгэх
manager = DataCollectionManager(config)

# Facebook ба Twitter-ээс сүүлийн 7 хоног
# "AI", "data science" түлхүүр үгүүдээр цуглуулалт
end_date = datetime.now()
start_date = end_date - timedelta(days=7)

results = manager.collect_from_platforms(
    platforms=[Platform.FACEBOOK, Platform.TWITTER],
    keywords=["AI", "data science"],
    start_date=start_date,
    end_date=end_date,
    max_posts=500,
    max_comments_per_post=100
)

# Цуглуулсан өгөгдлийн зургалалт авах
summary = manager.get_collection_summary()
print(f"Total posts collected: {summary['total_posts']}")
print(f"Total comments: {summary['total_comments']}")

# Өгөгдлийг JSON форматаар экспортлох
data = manager.export_data()
\end{verbatim}

Энэхүү модуль нь мультиплатформ цуглуулалтыг жигд интерфейсээр дамжуулж, судалгаанд ашигладаг ахиц онтолог, уян хатан цагүүлэхүйн болон түлхүүр үг шүүлтэр үйлчилгээг олголоор бүрдүүлсэн.

\section{Frontend Админ Хяналт ба Mock Өгөгдөл}

Системийн frontend хэсэг нь React + Vite суурьтайгаар хэрэгжсэн бөгөөд админ удирдлагын хуудсанд (AdminControl) цуглуулалтын тестийн query ажиллуулах таб нэмсэн. Энэхүү таб нь платформ, түлхүүр үг, эхлэх/дуусах огноо гэсэн параметрүүдээр өгөгдөл шүүх интерфейсээр хангана.

Хэрэгжилтийн одоогийн үе шатанд query ажиллагааг backend рүү шууд дуудахгүйгээр \texttt{mockCollectedPosts} жагсаалт дээр шүүлт хийж үр дүнг харуулж баталгаажуулсан. Шүүлтийн логик нь дараах гурван шалгуурыг хамтад нь мөрддөг:

\begin{enumerate}
    \item Платформын шүүлт (facebook/twitter/instagram/all)
    \item Огнооны шүүлт ($start\_date \leq post.date \leq end\_date$)
    \item Түлхүүр үгийн шүүлт (content болон keywords талбарт case-insensitive хайлт)
\end{enumerate}

Үр дүнг админ хуудсанд \texttt{Matched posts} хэмжүүр болон эхний багц постын жагсаалтаар харуулсан нь системийн query интерфейсийг backend интеграци хийхээс өмнө UI түвшинд баталгаажуулахад ашиглагдсан.

\section{Gemini API Тохиргоо (Vite Environment)}

Predictive analysis хэсэгт Gemini API ашиглахдаа frontend орчноос \texttt{import.meta.env.VITE\_GEMINI\_API\_KEY} хувьсагчийг уншиж \texttt{GoogleGenAI} клиентийг инициализац хийдэг. Хэрэв энэхүү хувьсагч тохируулаагүй бол систем fallback мессеж буцааж, AI шинжилгээг зогсооно.

BERT болон TextBlob ашиглан мэдрэмжийн дүн шинжилгээ хийнэ:

Нийтлэлүүдийн харилцан үйлдлийн гүнзгий анализ:

Engagement analysis хэсэг нь likes, shares, comments зэрэг үзүүлэлтүүдийг нэгтгэн пост тус бүрийн нөлөөллийн оноо, харьцангуй идэвхийг тооцоолдог. Энэ үзүүлэлт нь реакцын хэмжээний таамаглалд гол оролтын шинж болж ашиглагдсан.

\section{Үр Дүнгийн Хэлэлцүүлэг}

\textbf{RQ1-ийн хүрээнд:} BERT-д суурилсан Sentiment Analysis 91\% нарийвчлал үзүүлсэн нь H1 таамаглалыг дэмжиж байна.

\textbf{RQ2-ийн хүрээнд:} Мэдрэмж, оролцоо, хугацааны шинжүүдийг хамтатгасан үед хариу үйлдлийн чиглэлийн таамаглал Accuracy 0.84, Macro F1 0.81 болсон нь нэг шинжид суурилсан аргуудаас илүү үр дүнтэй байгааг илтгэнэ.

\textbf{RQ3-ийн хүрээнд:} Реакцын хэмжээний таамаглал MAE 18.6, RMSE 27.4 гарсан нь контент төлөвлөлт, нийтлэх хугацаа сонгох шийдвэрт ашиглаж болох түвшний алдаа хүлцэлтэй байгааг харуулж байна.

Ингэснээр уг систем нь зөвхөн техникийн хэрэгжилт бус, судалгааны зорилго болох мэдрэмжийн шинжилгээ ба хариу үйлдлийн таамаглалыг эмпирик байдлаар баталсан үр дүнтэй байна.
